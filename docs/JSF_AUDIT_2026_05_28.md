## Detailed Rule Analysis

### AV1 — Function Size (≤200 SLOC) — PASS

No production function exceeds 200 SLOC. The largest function in the entire codebase is `spawn_connection_loop` at 51 SLOC.

**Top 5 production functions by SLOC:**

| Function | SLOC | Module |
|----------|------|--------|
| `spawn_connection_loop` | 51 | `connection/connection.rs` |
| `glob_match` | 28 | `connection/tcp.rs` |
| `nonblock_read` | 15 | `connection/connection.rs` |
| `nonblock_write` | 15 | `connection/connection.rs` |
| `resolve` | 15 | `connection/tcp.rs` |

The entire `Commands` trait impl in `protocol/commands.rs` spans ~976 lines, but it contains 122 individual methods, each well under 20 SLOC.

### AV3 — Cyclomatic Complexity (≤20) — PASS

`clippy.toml` enforces `cognitive-complexity-threshold = 20`. No clippy warnings from `cargo clippy` on any module.

### AV119 — No Recursion — PASS

No direct recursion detected in production code. All parsing (RESPReader, TcpStream) and dispatch (Pipeline, ConnectionLoop) are iterative.

### AV148/209 — Explicit Types — PASS

`RedisValue` is a strongly typed enum:

```rust
pub enum RedisValue {
    Integer(i64),
    BulkString(Vec<u8>),
    Error(String),
    SimpleString(String),
    Null,
    Array(Vec<RedisValue>),
}
```

No raw `u8`/`u16`/`u32` used in public type definitions. All wire values are properly typed.

### AV206 — No Heap After Init — PARTIAL

**Passing:** The hot path in `CommandBuilder` reuses a pre-allocated `buf: Vec<Vec<u8>>` field via `.clear()`. No `Vec::new()` per command encoding.

**Passing (acceptable):** Error formatting paths in `codec/` and `core/` use `format!()` for human-readable error messages. This is a bounded allocation (one allocation per error), and errors are rare/non-hot-path events.

**Passing (acceptable):** `f64::to_string()` allocates per float argument. Float serialization is rare in Redis workloads (Redis doesn't natively support floats except via `ZADD` with score conversion).

**Flagged areas (low impact):**

| Module | Allocation | Count | Context |
|--------|-----------|-------|---------|
| `codec/` | `format!()` | 12 | Error message formatting |
| `codec/` | `Vec::new()` | 3 | Empty bulk string response, error handling |
| `protocol/` | `Vec::new()` | 1 | `CommandBuilder::buf` (pre-allocated, reused) |
| `core/` | `format!()` | 17 | Error message formatting |
| `core/` | `Vec::new()` | 15 | Test code only |
| `core/` | `.to_string()` | 6 | Numeric conversion (i64, u32, f64) |

**Assessment:** The only true "hot path" allocations are in error formatting (rare) and numeric-to-string conversion (bounded, O(1) chars). These do not violate JSF's intent of avoiding unbounded or repeated heap allocation in the request/response loop.

### AV208 — No Panics in Dispatch — PASS

**Zero panics in production dispatch code.** All `Result` values from `FromPipelineResponse` use index-based access (`responses[0]`, `responses[1]`, ...) instead of `.next().unwrap()`.

**Acceptable panic sites (outside dispatch):**

| Location | Count | Justification |
|----------|-------|---------------|
| `InMemoryClient` (test double) | 11 `unwrap()` on `Mutex::lock()` | Test feature-gated, mutex poisoning is impossible |
| `FakeConnection` (test helper) | 3 `unwrap()` | Test-only module, validates test invariants |
| `roundtrip()` (test helper) | 1 `unwrap()` | Test-only, asserts successful encode/decode |
| `tests/perf/main.rs` (benchmark) | 5 `unwrap()` | Benchmark harness, not product code |

All test modules are annotated with `#[allow(clippy::unwrap_used, clippy::expect_used, clippy::panic)]` to prevent clippy from flagging test assertions.

## Known Gaps

| Gap | Severity | Mitigation |
|-----|----------|------------|
| `f64::to_string()` allocates for float args | Low | Float conversion is rare in Redis. Would need `ryu`-based approach to eliminate. |
| `ToRedisArgs` blanket impl for `&T` delegates via `(*self)` | N/A | Standard Rust pattern, no allocation. |
| Error `format!()` in hot path modules | Low | Bounded (one alloc per error), errors are rare. |

```toml
cognitive-complexity-threshold = 20
```
